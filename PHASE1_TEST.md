# Phase 1 测试方法文档

## 1. 测试环境准备

### 1.1 编译

```bash
cd /home/baiyf/RustLuaMud
cargo build --release
```

二进制文件位于 `target/release/rust-lua-mud`

### 1.2 测试服务器

使用真实 MUD 服务器进行测试：

- **地址**: `ln.xkxmud.com`
- **端口**: `5555`
- **编码**: GBK

> 注意：该服务器使用 GBK 编码，客户端已内置 GBK→UTF-8 自动转码。
> 测试时请确保网络可访问该服务器。

### 1.3 配置文件

修改 `configs/default.toml`：

```toml
[general]
scroll_buffer = 5000
log_dir = "logs"
log_rotation_size_mb = 10
log_rotation_count = 5

[[connections]]
name = "侠客行"
host = "ln.xkxmud.com"
port = 5555
auto_connect = true
auto_reconnect = true
reconnect_delay_secs = 5
```

## 2. 功能测试用例

### 测试 1: 启动与配置加载

**步骤：**
1. 确保 `configs/default.toml` 存在且配置正确
2. 运行 `./target/release/rust-lua-mud`

**预期结果：**
- 终端切换到 TUI 模式（备用屏幕缓冲区）
- 状态栏显示：`[1]侠客行 ● 1/1 RustLuaMud`
- 输出区显示：`[系统] 连接 1 (侠客行) 已建立`
- 底部输入行显示绿色 `> ` 提示符和闪烁光标

**异常测试：**
- 删除 `configs/default.toml`，启动应显示警告但使用默认配置（无连接）
- 配置文件中 host 错误，启动应显示连接失败信息

### 测试 2: TCP 连接与 GBK 编码接收

**步骤：**
1. 启动 rust-lua-mud
2. 观察服务器欢迎信息

**预期结果：**
- 输出区正确显示中文欢迎信息（GBK 自动转 UTF-8）
- 无乱码、无问号方块
- ANSI 转义序列被正确剥离（不显示乱码控制字符）
- 多行文本逐行显示

**关键验证点：**
- GBK 中文字符（如"侠客行"、"欢迎"）正确显示
- GBK 特殊标点（全角符号）正确显示
- 混合 ANSI 颜色码 + GBK 文本时，颜色码被剥离、中文正常

### 测试 3: 命令发送

**步骤：**
1. 在输入行输入 `look`
2. 按 Enter

**预期结果：**
- 输出区回显 `> look`
- 服务器收到命令并回复场景描述
- 回复内容（GBK 编码）正确转码显示
- 输入行清空，光标回到行首

**额外测试：**
- 输入中文命令（如果服务器支持），验证 UTF-8→GBK 发送转码正确
- 输入简单英文命令如 `hp`、`score`、`chat hello`

### 测试 4: 输入行编辑

**步骤：**
1. 输入 `look`，然后按左箭头将光标移到 `l` 和 `o` 之间
2. 输入 `x`，观察变为 `lxook`
3. 按 Backspace，观察变为 `look`
4. 按 Home，光标到行首
5. 按 End，光标到行尾
6. 按 Delete（光标在行首时），删除 `l`

**预期结果：**
- 左右箭头正确移动光标
- 字符插入在光标位置
- Backspace 删除光标前字符
- Delete 删除光标后字符
- Home/End 正确跳转

### 测试 5: 命令历史

**步骤：**
1. 依次输入并发送 `look`、`hp`、`score`
2. 按上箭头，应显示 `score`
3. 再按上箭头，应显示 `hp`
4. 再按上箭头，应显示 `look`
5. 按下箭头，应显示 `hp`
6. 再按向下箭头，应显示 `score`
7. 再按向下箭头，输入行清空

**预期结果：**
- 上下箭头正确浏览历史
- 选中历史条目后可直接发送或编辑

### 测试 6: 断线检测

**步骤：**
1. 连接正常后，断开网络（或等待服务器维护重启）
2. 观察客户端行为

**预期结果：**
- 输出区显示 `[系统] 连接 1 (侠客行) 已断开`
- 状态栏图标变为 `○`
- 显示 `[系统] 5 秒后尝试重连 侠客行...`

### 测试 7: 自动重连

**步骤：**
1. 断线后，等待自动重连
2. 或在服务器维护结束后观察重连行为

**预期结果：**
- 客户端自动重新连接
- 状态栏恢复为 `●`
- 收到新的欢迎信息
- GBK 编码再次正确转码

### 测试 8: 退出

**步骤：**
1. 按 Ctrl+C 或 Ctrl+D

**预期结果：**
- 程序立即退出
- 终端恢复正常模式（回到命令行）
- 无残留的 raw mode 或备用屏幕缓冲区

### 测试 9: 终端大小变化

**步骤：**
1. 调整终端窗口大小
2. 观察输出区和输入行是否正确重绘

**预期结果：**
- 状态栏、输出区、输入行正确适应新尺寸
- 无显示错乱

### 测试 10: GBK 编码边界测试

**步骤：**
1. 连接服务器后，观察各种场景输出
2. 特别注意包含以下内容的输出：
   - 中文标点（，。！？；：""''）
   - 特殊符号（【】《》、）
   - 混合中英文的行
   - 纯英文行
   - 服务器状态栏（通常包含 ANSI 颜色 + 中文）

**预期结果：**
- 所有 GBK 字符正确转码为 UTF-8
- 无截断（GBK 双字节字符不被从中间拆开）
- 无乱码

## 3. 稳定性测试

### 3.1 长时间运行测试

**步骤：**
1. 连接到 `ln.xkxmud.com:5555`
2. 让客户端保持连接 4 小时以上
3. 定期检查内存占用

```bash
# 另一个终端监控内存
watch -n 60 'ps aux | grep rust-lua-mud | grep -v grep'
```

**预期结果：**
- RSS 内存占用稳定，不持续增长
- 连接保持正常
- 无 panic 或崩溃
- GBK 转码无内存泄漏

### 3.2 高频数据测试

**步骤：**
1. 在 MUD 中进入人多的区域（如城市中心），触发大量聊天和场景输出
2. 观察客户端是否流畅

**预期结果：**
- 输出区持续更新，无明显卡顿
- 输入行仍可正常输入和发送命令
- 内存不持续增长
- GBK 转码在高频下不丢字符

### 3.3 断线重连测试

**步骤：**
1. 手动断开网络连接（如 `sudo ifconfig eth0 down`）
2. 等待 30 秒后恢复网络（`sudo ifconfig eth0 up`）
3. 观察客户端重连行为

**预期结果：**
- 断线被正确检测
- 网络恢复后自动重连成功
- 重连后 GBK 编码仍正确工作
- 无 panic、无死锁、无内存泄漏

## 4. 性能基准

在 J1800 + 2GB 环境上的目标指标：

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 启动时间 | < 2 秒 | `time ./target/release/rust-lua-mud` |
| 单连接内存占用 | < 10MB | `ps aux` 查看 RSS |
| CPU 空闲占用 | < 1% | `top` 查看，连接空闲时 |
| CPU 活跃占用 | < 5% | `top` 查看，高频数据时 |
| 输入延迟 | < 50ms | 主观感受，按键到字符显示 |
| GBK 转码延迟 | 不可感知 | 与纯英文输出对比无差异 |

## 5. 已知限制（Phase 1）

- ANSI 颜色被剥离而非渲染（Phase 2 完善）
- 仅前台连接渲染，后台连接数据暂不记录日志（Phase 4 完善）
- 自动重连逻辑尚未实现定时器触发（Phase 4 完善）
- 无 Lua 脚本支持（Phase 3 完善）
- 无滚动回看功能（PageUp/PageDown）
- 发送命令暂以 UTF-8 编码，中文命令需确认服务器是否支持（大部分 MUD 服务器仅接受 ASCII 命令）
